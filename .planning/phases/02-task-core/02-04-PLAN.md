# Phase 02 Plan 04: Task Presentation — Core Screens and Widgets

## Goal

Implement the five core task screens (task list, project, task form, Eisenhower board, 1-3-5 day planner) and four shared widgets, all wired to the cubits from Plan 03, with a visual verification checkpoint after implementation.

## Context

Plans 01-03 deliver the domain, data, and application layers. This plan adds the presentation layer: screens that consume cubits via `BlocBuilder`/`BlocConsumer`, forms that call cubit methods, and a 2x2 Eisenhower board and 1-3-5 planner built from Material 3 components. No new packages — all UI uses Flutter SDK widgets.

All UI text must use the l10n system (`context.l10n.*` or `AppLocalizations.of(context)!.*`) — no hardcoded English strings (CLAUDE.md: UI text in PT-BR with EN toggle). ARB key additions for new strings go in both `lib/l10n/app_en.arb` and `lib/l10n/app_pt_BR.arb`.

### Wave 1 — Shared Widgets

#### Task 2: Shared task presentation widgets
**Files:**
- `lib/presentation/tasks/widgets/task_card.dart`
- `lib/presentation/tasks/widgets/quadrant_card.dart`
- `lib/presentation/tasks/widgets/slot_section.dart`

**Action:**

`lib/presentation/tasks/widgets/task_card.dart`:

A reusable card for a single `Item`. Props: `item`, `onComplete`, `onDelete`, `onTap`. Renders:
- Title (text, one line, overflow ellipsis)
- Priority chip (colored badge using `Chip` with `backgroundColor` from priority color map)
- Due date label (if `item.dueDate != null`) formatted as `dd/MM/yyyy`
- Eisenhower quadrant label (use `item.eisenhowerQuadrant.name` for now — l10n label in Plan 05)
- Checkbox for `isCompleted`
- Delete icon button that calls `onDelete`

Priority color map (use `colorScheme` tokens):
- `Priority.urgent` → `colorScheme.error`
- `Priority.critical` → `colorScheme.errorContainer`
- `Priority.high` → `colorScheme.tertiaryContainer`
- `Priority.medium` → `colorScheme.secondaryContainer`
- `Priority.low` → `colorScheme.surfaceVariant`

Do NOT import Isar in this widget. Import only `package:agenda/domain/tasks/item.dart` and `package:agenda/domain/tasks/priority.dart`.

`lib/presentation/tasks/widgets/quadrant_card.dart`:

A card for one Eisenhower quadrant. Props: `label` (String), `items` (List<Item>), `colorScheme` color for the header. Renders:
- Header with colored background and label text
- Scrollable list of task title chips inside the card body
- "Vazia" (empty state text) when items is empty

`lib/presentation/tasks/widgets/slot_section.dart`:

A section for one 1-3-5 Rule slot. Props: `label` (String), `maxSlots` (int from AppConstants), `currentItems` (List<Item>), `isOverCapacity` (bool), `onTapAdd` (VoidCallback), `onRemove` (void Function(int id)).

Renders:
- Section header: `"$label ($currentCount/$maxSlots)"`
- Warning banner (amber/yellow) when `isOverCapacity == true` — text: "Limite excedido" / "Limit exceeded"
- List of assigned items with a remove icon button each
- An "Adicionar" / "Add" action that calls `onTapAdd`

#### Task 4: Task form screen
**Files:**
- `lib/presentation/tasks/screens/task_form_screen.dart`

**Action:** A single screen used for both create and edit. Pass `item` as optional constructor param — null means create mode; non-null means edit mode.

Fields (in order):
1. **Title** — `TextFormField`, required, validator: `if (value.isEmpty) return l10n.titleRequired`
2. **Description** — `TextFormField`, optional, maxLines 3
3. **Type** — `SegmentedButton<ItemType>` with segments for `project`, `task` (subtask not selectable here — subtasks are created via `ProjectCubit.addSubtask`)
4. **Priority** — `DropdownButtonFormField<Priority>` with all 5 values
5. **Due Date** — `ListTile` with `trailing: TextButton` that calls `showDatePicker`; displays the selected date or "Sem prazo" / "No due date"
6. **Due Time** — `ListTile` with `trailing: TextButton` that calls `showTimePicker` (only enabled if dueDate is set); displays HH:mm or "Sem hora" / "No time"
7. **Urgente / Urgent** — `SwitchListTile` for `isUrgent`
8. **Importante / Important** — `SwitchListTile` for `isImportant`
9. **Tamanho / Size** — `SegmentedButton<SizeCategory>` with big/medium/small/none segments
10. **Próxima ação / Next action** — `CheckboxListTile` for `isNextAction`
11. **Contexto GTD / GTD Context** — `TextFormField` optional, hint "@home"
12. **Aguardando / Waiting for** — `TextFormField` optional
13. **Valor / Amount** — `TextFormField` optional, `keyboardType: TextInputType.numberWithOptions(decimal: true)`
14. **Moeda / Currency** — `TextFormField` optional, hint "MZN", maxLength 3

Form state managed with `_formKey = GlobalKey<FormState>()` and local state variables in `StatefulWidget`. On save:
- Validate form: `if (!_formKey.currentState!.validate()) return;`
- Build `Item` from form fields with `createdAt = DateTime.now()`, `updatedAt = DateTime.now()` for create; `item.copyWith(updatedAt: DateTime.now())` for edit
- Call `context.read<TaskListCubit>().createItem(item)` (add `createItem` to `TaskListCubit`) or `updateItem`
- Pop the route

Add `createItem` method to `TaskListCubit`:
```dart
Future<void> createItem(Item item) async {
  final result = await _repository.createItem(item);
  if (result is Err) emit(TaskListError((result as Err).failure));
  // watchChanges() fires reload automatically
}
```

Also add `updateItem` method:
```dart
Future<void> updateItem(Item item) async {
  final result = await _repository.updateItem(item);
  if (result is Err) emit(TaskListError((result as Err).failure));
}
```

Store `dueTimeMinutes` as `selectedTime.hour * 60 + selectedTime.minute`.

## Threat Model

| Threat ID | Description | Severity | Disposition | Mitigation |
|-----------|-------------|----------|-------------|------------|
| T-02-01 | Isar query injection via user search string | medium | mitigate | Search input passed to TaskListCubit.search() which delegates to repository.searchByTitle() — typed API, no interpolation |
| T-02-05 | Soft-deleted items visible via stale Cubit state | medium | mitigate | TaskListScreen BlocConsumer rebuilds on every state change; TaskListWithPendingUndo state excludes deleted item immediately |
| T-02-07 | Form submits with empty title due to missing validator | low | mitigate | TextFormField validator returns error string on empty; Form.validate() checked before createItem/updateItem call |

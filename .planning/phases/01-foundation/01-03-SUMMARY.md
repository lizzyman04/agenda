# Phase 01 Plan 03: GetIt + Injectable DI Graph Summary

**One-liner:** Four-module injectable DI graph (CoreModule + three domain placeholders) with IsarService singleton, SharedPreferences preResolve, and configureDependencies() wired in main.dart before runApp.

## What Was Built

A fully working GetIt + injectable dependency injection graph split into four domain modules:

1. **`lib/config/di/injection.dart`** — global `getIt` instance + `configureDependencies()` entry point annotated with `@InjectableInit()`
2. **`lib/config/di/core_module.dart`** — `@module` registering `IsarService` as `@singleton` and `SharedPreferences` as `@preResolve` async singleton
3. **`lib/config/di/tasks_module.dart`** — empty `@module` placeholder for Phase 2 Task domain registrations
4. **`lib/config/di/finance_module.dart`** — empty `@module` placeholder for Phase 3 Finance domain registrations
5. **`lib/config/di/infrastructure_module.dart`** — empty `@module` placeholder for Phase 4-5 infrastructure services
6. **`lib/config/di/injection.config.dart`** — generated by `dart run build_runner build`; registers `SharedPreferences`, `IsarService`, and `LocaleCubit` (auto-discovered from Plan 04 code)
7. **`lib/main.dart`** — updated to `await configureDependencies()` before `runApp()`
8. **`test/config/di/di_test.dart`** — 3 smoke tests confirming GetIt graph resolves correctly

## Generated injection.config.dart

The generator auto-discovered `LocaleCubit` (from Plan 04) alongside the CoreModule registrations:

```dart
// SharedPreferences (preResolve — awaited at startup)
await gh.factoryAsync<SharedPreferences>(() => coreModule.sharedPreferences, preResolve: true);

// IsarService (singleton — single Isar connection)
gh.singleton<IsarService>(() => coreModule.isarService);

// LocaleCubit (factory — auto-discovered from locale_cubit.dart)
gh.factory<LocaleCubit>(() => LocaleCubit(gh<SharedPreferences>()));
```
